# Микросервис аналитики (backend)

Микросервис отвечает за:
- работу с датасетом компаний и товарных знаков
- расчёт аналитических показателей 
- фильтрацию и агрегацию данных
- предоставление REST API для дашборда
- интеграцию модели поиска похожих логотипов через CV-модель

## Основные файлы:
main.py — API
data_loader.py — работа с данными
model_adapter.py — интеграция с моделью

## Модуль работы с данными — data_loader.py. отвечает за:
- загрузку CSV-датасетов
- нормализацию данных
- обработку классов МКТУ
- фильтрацию компаний
- получение статистик

## CV-интеграция (поиск логотипов) ¬— model_adapter.py. Модуль обеспечивает:
- валидацию изображения
- запуск модели
- обработку результатов
- возврат top-k логотипов
- возврат оценки риска

## Модуль работы с МКТУ — файлы parsing_mktu.py, json_2.py, intersection_json.py, result_csv_mktu.py. Отвечает за:
- парсинг МКТУ
- повторный сбор пропущенных данных
- объединение результатов
- формирование CSV

## Тестирование — папка tests/

Содержит unit-тесты:
- API
- работа с данными
- обработка изображений

# Основная логика работы

1. Подготавливаются данные
2. Парсятся классы МКТУ
3. Формируется датасет
4. Микросервис загружает данные
5. Пользователь делает запрос
6. Выполняется аналитика или поиск
7. Возвращается результат
# Рекомендуется следующий порядок запуска: 
1.	установить зависимости проекта (pip install -r requirements.txt)
2.	запустить микросервис (если локально — uvicorn main:app –reload)
3.	проверить, что модуль компьютерного зрения доступен микросервису
4.	запустить дашборд
5.	открыть интерфейс в браузере
## Запуск микросервиса 
Он должен быть запущен первым, так как дашборд получает данные через HTTP-запросы к API. В проекте микросервис реализован как REST-сервис, поэтому его запуск выполняется отдельно. Запуск может быть выполнен с помощью команды: 
nohup venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000 > nohup.out 2>&1 &
## Запуск модуля компьютерного зрения
Этот модуль используется для поиска похожих логотипов по изображению. В проекте он интегрирован в микросервис и вызывается при выполнении запроса. В случае интеграции через микросервис отдельный ручной запуск не требуется, но необходимо, чтобы пути к модели, данным и выходным JSON-файлам были корректно настроены. 
## Запуск дашборда 
Команда запуска:
streamlit run app.py
После этого локальный сервер Streamlit открывает веб-страницу в браузере.
